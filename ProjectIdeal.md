Pantry-to-Plate" — Smart Meal Planning Ecosystem
1.Home / Dashboard — today's planned meal, expiring pantry items, shopping list summary, weekly nutrition snapshot
2.Pantry — inventory of ingredients on hand (quantity, expiration date, category)
Recipe Discovery — search recipes (Spoonacular API), filtered/ranked by what's already in your Pantry ("you can make this with 80% of what you have")
3.Meal Planner — drag recipes onto a weekly calendar; pulls directly from Recipe Discovery
Shopping List — auto-generated by comparing Meal Planner's required ingredients against current Pantry stock (computed, not manually entered)
4.Nutrition Dashboard — aggregates nutritional data from the week's planned meals (calories, macros) via charts
5.Cooking Mode — step-by-step recipe view; on completion, auto-deducts used ingredients from Pantry (closing the loop back to step 2)
6.Settings/Profile — dietary preferences (vegetarian, allergies, etc.) that filter Recipe Discovery and Nutrition targets globally
Why this satisfies "meaningful ecosystem"
The core sell is this: Pantry → Recipes → Meal Plan → Shopping List → Cooking → back to Pantry is a closed data loop. No module is decorative — each one depends on real output from another, which is what your instructor is asking for instead of standalone pages sharing a navbar.
 "Each module consumes the previous module's output and produces input for the next, forming a closed loop rather than independent features.

*Project Title: `QuizForge` – A Full-Stack React Quiz Platform*

*1. OVERVIEW* 
*QuizForge is a responsive React web app that tests users across 5 tech topics. Its a full React + API + Database project, deployed on Netlify + GitHub.*

*2. CORE FEATURES* 
- *Auth + Dashboard*: *User signup/login with Supabase Auth. Personal dashboard after login.*
- *Quiz Engine*: *3 difficulty modes → Easy: 20 Qs, Medium: 30 Qs, Hard: 50 Qs. Timer per question.*
- *5 Topics via API*: *MongoDB, Java, HTML, CSS, JavaScript, React. Questions fetched from Open Trivia DB API or custom Supabase table.*
- *Scoring + Streaks*: *+10 per correct. Streak multiplier x1.5. Timer penalty for slow answers.*
- *Routing*: *React Router: `/login`, `/dashboard`, `/quiz/:topic/:level`, `/leaderboard`, `/stats`*
- *Leaderboard*: *Global + Class ranks saved in Supabase. Updates after each quiz.*

*3. COMPLEXITY FEATURES* 
- *PWA Offline Mode*: *Works without internet using IndexedDB + Workbox. Syncs scores when online.*
- *Instant Explanations*: *After each Q, show AI-generated explanation. No chatbot needed.* 
  *Logic: If wrong → "Correct answer is B. Reason: [AI explanation]" like Duolingo.*
- *Deployment*: *GitHub for code + version control. Netlify for live deploy with CI/CD.*

*4. TECH STACK* 
*Frontend: React + Vite + Tailwind + React Router + Zustand* 
*Backend: Supabase = Auth + Database + API* 
*API: Open Trivia DB or custom JSON API for questions* 
*AI: OpenAI API or free HuggingFace model for 1-sentence explanations* 
*Deploy: GitHub → Netlify*
